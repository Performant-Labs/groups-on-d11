do_activity_feed — activity feed wireframe (issue #129)

Low-fidelity mockup of /activity (my_groups scope, persona Elena) covering the six row shapes from the brief interleaved in one feed, plus the empty state. Matches the visual language of the approved do_streams shell wireframe (#109) — same tokens, same .shell/.card/.gc-empty conventions — rather than inventing a new one. Static HTML/CSS only; the <details>/<summary> disclosure is the only interactive behavior and is native (no JS).

Route note. This wireframe shows /activity (my_groups scope, the default/only scope for this story per the brief — there is no "Following"/"Trending" scope for activity, unlike do_streams). The group-scope variant /activity/group/{group} renders the identical row shapes filtered to one group; it is not drawn twice here — see the linked note under the scope-tab strip in State 1 below. "Trending" tab shown is illustrative of future ST-8 mounting, not a working control in this story (see open question).

Maps to:
  • activity-row-content — wraps node.stream_card render output (black-boxed here, owned by the existing node view-mode template) with the compact meta strip card__meta: "Posted by [actor] in [group], [time]". Actor/group/title are links; snippet is the node's existing teaser text.
  • activity-row-social (used for join / RSVP / comment / group-created) — one-line sentence built from actor + verb + target + group, timestamp trailing. Comment rows add a truncated (≤180 char, tags-stripped) italic quote line beneath the sentence, per the kernel acceptance criterion.
  • activity-row-aggregated — native <details>/<summary>; the <summary> IS the aggregated sentence including the actor link, group link, and the textual count ("3 topics" — non-color status signal). Expanding reveals the underlying titles as links (children render-array data). Keyboard-operable natively (Tab to focus, Enter/Space to toggle); the disclosure triangle glyph (▸) is a Unicode character, not hand-drawn SVG, and rotates via CSS on [open].
  • All actor names, group names, node/comment/event titles are anchors (<a>) — real hrefs in production, inert # in this static mock.
  • Left-gutter avatar circle (row__avatar) is shared across social/aggregated rows for visual rhythm; content-card rows omit it since the card itself carries its own author line (matches how do_streams' stream_card already renders authorship — no duplication).
